====================================================================================================
GNN排名分析報告說明
====================================================================================================

本資料夾包含兩種不同的GNN排名分析報告：

1. 原始排名報告 (gnn_only_ranking_*.txt)
   - 排名範圍: 1-118
   - 說明: 每個GNN變體在所有118個競爭者中的排名
   - 競爭者包含:
     * 9個模型 × 6個配置 (1 none + 5 GNN stages) × 2個ratio = 108
     * 5個參考模型 × 2個ratio = 10
     * 總計: 118個競爭者
   - 用途: 了解GNN變體相對於所有模型（包括原始模型和參考模型）的絕對表現

2. 相對排名報告 (gnn_relative_ranking_*.txt)
   - 排名範圍: 1-45
   - 說明: 只在45個GNN變體之間重新計算的相對排名
   - 競爭者包含:
     * 9個模型 × 5種GNN插入階段 = 45個GNN變體
     * 排除: none階段（原始模型）和參考模型
   - 用途: 直接比較不同GNN插入策略的相對優劣

====================================================================================================
關鍵差異示例 (large_datasets+binclass+numerical 分類)
====================================================================================================

配置: Train/Val/Test = 0.05/0.15/0.8 (小訓練集)

原始排名（在118個競爭者中）:
  1. trompt(columnwise)      平均排名: 25.33
  2. trompt(encoding)         平均排名: 27.33
  3. resnet(columnwise)       平均排名: 36.00
  
  解讀: trompt(columnwise)在118個競爭者中平均排在第25-26名
        這意味著它超越了約79%的競爭者（93/118）

相對排名（只在45個GNN變體中）:
  1. trompt(encoding)         平均排名: 2.17
  2. trompt(columnwise)       平均排名: 2.67
  3. resnet(columnwise)       平均排名: 6.17
  
  解讀: trompt(encoding)在45個GNN變體中平均排在第2名
        這意味著它是最好的GNN增強策略之一

====================================================================================================
使用建議
====================================================================================================

1. 如果要評估GNN增強是否有效（與原始模型和參考模型相比）：
   → 使用 gnn_only_ranking_*.txt
   → 查看GNN變體能否進入前20名（相對於118個競爭者）

2. 如果要選擇最佳的GNN插入階段和模型組合：
   → 使用 gnn_relative_ranking_*.txt
   → 專注於45個GNN變體之間的相對表現
   → 更容易看出哪個stage和哪個模型的組合最優

3. 關鍵發現：
   - 大訓練集 (0.8/0.15/0.05): resnet和trompt的encoding/columnwise階段表現最好
   - 小訓練集 (0.05/0.15/0.8): trompt的encoding/columnwise階段表現最好
   - 不同數據集分類下的最佳GNN配置有所不同

====================================================================================================
技術細節
====================================================================================================

原始排名計算方式:
  1. 對每個數據集，所有118個競爭者一起排名
  2. 每個GNN變體獲得一個排名（1-118）
  3. 計算每個GNN變體在同一分類所有數據集上的平均排名

相對排名計算方式:
  1. 對每個數據集，提取45個GNN變體的原始排名
  2. 根據原始排名的大小關係，重新分配排名（1-45）
  3. 計算每個GNN變體在同一分類所有數據集上的平均相對排名

數據集分類:
  - 按大小: large_datasets, small_datasets
  - 按任務: binclass, multiclass, regression
  - 按特徵: numerical, categorical, balanced (混合)
  - 總計: 10種分類組合，116個數據集
